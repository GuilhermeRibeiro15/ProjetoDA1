Prompts the user for a station name and displays information about the outgoing and incoming tracks of the station. 

The function first asks the user to enter a station name. If the name is \char`\"{}\+Quit\char`\"{}, the function returns. Otherwise, it searches for the station in the hashtable using the find\+Station\+In\+Hashtable function. If the station is not found, the function informs the user and prompts them to enter the name again. If the station is found, the function displays the station name and a list of outgoing tracks followed by a list of incoming tracks. For each track, the function displays the destination/origin station name, capacity, and service. {\footnotesize\ttfamily int Trip\+Manager\+::ler\+Ficheiros (\begin{DoxyParamCaption}\item[{string}]{station\+File,  }\item[{string}]{track\+File }\end{DoxyParamCaption})}



This function reads the stations and tracks data from two CSV files and creates the corresponding objects. 

It opens and reads the \char`\"{}stations.\+csv\char`\"{} file and creates a \mbox{\hyperlink{class_station}{Station}} object for each line, then adds it to the graph data structure. Then, it opens and reads the \char`\"{}network.\+csv\char`\"{} file and creates a \mbox{\hyperlink{class_track}{Track}} object for each line, linking the corresponding \mbox{\hyperlink{class_station}{Station}} objects. The function also adds the newly created \mbox{\hyperlink{class_station}{Station}} objects to a hash table for easier lookup. Time complexity\+: O(V log V + E log V), where V is the number of stations and E is the number of tracks in the input files. \begin{DoxyNote}{Note}
If either file cannot be opened, the function prints a \char`\"{}\+File not found\char`\"{} message and returns. 
\end{DoxyNote}
